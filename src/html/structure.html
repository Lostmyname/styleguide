Structure

Jester is comprised of a set of styling layers, each of which comprises of the same file structure.

The three layers are:

Project ->
    Theme ->
        Component

These layers give the order of inheritance, with a projectâ€™s styles overriding a theme, and a theme overriding the base styles of a component.

Projects and themes structure look very similar, and both contain components.

The file structure is based around SMACSS;

|--bases/
   |--font/_font.scss
   |--typography/_typography.scss
   |--form/_form.scss
   |--icon/_icon.scss
|--components/
   |--*
|--layouts/
   |--*
|--_variables.scss
|--style.scss

Bases are essentially your underlying styles that make cover default tag styling and common elements.
Components are reusable page elements, made up of HTML, CSS and JS.
Layouts are where you put reusable CSS used for positioning elements.
